01
What CWPP is — it protects the workload itself, not the cloud around it

Cloud Workload Protection Platform — security that lives with the running workload: it scans it, hardens it, and watches its behaviour as it executes. CSPM checks the cloud config; CWPP defends the code that runs. It follows the workload across any cloud, on-prem, and hybrid — wherever it runs.

Think of it as a bodyguard that travels with each VM, container & function — from the image on the shelf to the process in production.

02
The workloads it covers — one platform, four runtime shapes

🖥️ Virtual machines

Long-lived hosts & their OS — patch & harden the classic estate.

📦 Containers

Images & running containers — scanned before deploy, watched after.

☸️ Kubernetes

Pods, nodes & the cluster — protection with orchestration context.

λ Serverless

Ephemeral functions — short-lived, hard to see, still in scope.

Same defence, wherever the code runs — no matter how long it lives.

03
Build → runtime — shift-left in CI, then keep guarding it in production

Scanning before deploy catches known bugs; runtime protection catches what the scan couldn't — the live attack.

04

06
Agent vs agentless — most platforms run both, so nothing is left uncovered

🔌 Agent-based

A sensor inside the workload → deep runtime visibility: live process, memory & syscall activity, and the power to block. Cost: an agent to deploy & run.

📸 Agentless

Snapshot-based scanning → broad, fast coverage with nothing to install. Great for vulns & config, but no live runtime view.

Agentless finds it everywhere; agents watch it as it runs. You want both.

07
A guarded workload — defences wrapped around the running thing, alerting on drift

Scans harden it before it runs; a live baseline flags the moment it does something it never did before.

Where it fits

CWPP is the workload pillar of a CNAPP — bundled with its siblings into one platform that follows the attack path:

CWPP+ CSPM+ CIEM+ CDR

One vendor, one console — CWPP is the piece that actually lives with the code.

10
Watch out
🪫
Agent overhead & gaps — agents cost resources and may miss hosts you never instrumented.
💨
Ephemeral blind spots — serverless & short-lived pods vanish before an agent can see them.
🔀
Scan ≠ reality — the image you scanned may not be what actually runs. Tie results to running workloads.
11

MFA vs PASSKEYS

Not all logins are equal — from phishable codes to a cryptographic signature bound to the real site (the FIDO2 / WebAuthn upgrade)
CTF Farooq Ali
· security explainer series
v2026.06
01
The problem — a typed shared secret is the weak link

A password is a shared secret you type — so it can be phished, reused across sites, and dumped in a breach.

02
MFA = two or more different factors

🧠 Something you KNOW

A password or PIN — a secret in your head.

📱 Something you HAVE

A phone, an authenticator app, or a security key.

👆 Something you ARE

A biometric — fingerprint or face.

Two passwords isn't MFA — the factors must be of different kinds.

03
Not all MFA is equal
📩
Phishable — SMS OTP, TOTP codes, push prompts. A code you can read out can be relayed to a fake site.
🛡️
Phishing-resistant — FIDO2 / WebAuthn passkeys & security keys. Nothing to read out or replay.

Adding a second factor helps — but a phishable one can still be intercepted or fatigue-approved.

04
How phishable MFA breaks
🔁
OTP / TOTP relay — a proxy site captures the code and replays it in real time.
📲
SMS intercept — SIM-swap or network flaws divert the text.
😵
Push fatigue — spam the prompt until a tired user taps “approve”.
05
What a passkey is — a public / private key pair (FIDO2 · WebAuthn)

No shared secret exists — the private key stays in your device's secure hardware; the server only ever holds the matching public key.

06
How passkey sign-in works — challenge → sign → verify

Nothing secret is transmitted — only a signature over a one-time challenge, which the server checks with your public key.

07
Why passkeys resist phishing
🔗
Bound to the origin — the key is tied to the site's RP ID. A look-alike domain can't get a valid signature.
🤫
No code to read out — there's no OTP to relay and no shared secret to hand over.
🗄️
Breach-safe server — a stolen database leaks only public keys, which are useless to an attacker.
08
Synced vs device-bound

☁️ Synced

Roams across your devices via a provider keychain.Apple · Google · 1Password

🔐 Device-bound

A hardware security key — the private key never leaves it.YubiKey · TPM

Synced = convenient recovery · device-bound = highest assurance.

10
Watch out
🚪
Recovery is the new soft spot — account reset flows are where attackers now push. Harden them.
🪜
Keep a fallback — a lost device shouldn't lock you out; register a backup key or passkey.
🎣
Legacy MFA lingers — OTP relay & push-fatigue still work if a phishable factor is left enabled.
🏢
Attestation & policy — enterprises may require device-bound keys and attestation to trust a credential.
11
The takeaway
Passkeys replace the shared secret with a signature bound to the real site — phishing-resistant by design, not by user vigilance.
Challenge→ Unlock→ Sign→ Verify with public key

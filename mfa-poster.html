MULTI-FACTOR AUTH

Prove it's really you with two or more factors of different types — so a stolen password alone can't get in (and the strongest factors resist phishing entirely)

01
What MFA is — 2+ factors of different types

A second factor only helps if it's a different kind of proof — two passwords is still single-factor.

02
The three factor types — know · have · are

🧠 Something you KNOW

Password, PIN, passphrase. Cheap — but phishable, guessable, reusable.

📱 Something you HAVE

Phone, authenticator app, security key. A device only you hold.

🫆 Something you ARE

Fingerprint, face, iris. A biometric — usually unlocks the device.

MFA = combine at least two of these three categories.

03
The methods, weakest → strongest — climb the ladder

Any MFA beats a lone password — but move users up toward phishing-resistant factors.

04
Adaptive / step-up MFA

Risk-based — challenge only when the context is risky, not on every click:

new devicenew locationimpossible travelsensitive action

Low friction when safe · strong challenge when it counts.

05
Where to require MFA

Ideally everywhere — but never skip these high-value entry points:

admin / privilegedremote / VPNemailsensitive transactions

Admins & remote access are the crown jewels — key them first.

06
How attackers bypass MFA — codes & push can be defeated

📶 SIM-swap

Hijack your phone number, then receive your SMS OTP. Kills SMS as a factor.

🪝 Real-time phishing (AiTM)

A relay proxy sits between you & the real site, forwarding your OTP live.

🔔 Push fatigue / MFA bombing

Spam approval prompts until a tired user finally taps approve.

🎣 OTP phishing

Trick the user into typing a code straight into the attacker's page.

Anything the user can read or tap, an attacker can trick them into relaying.

07
Adversary-in-the-middle — how a code / push gets relayed

A passkey / security key breaks this at step 4 — the credential is bound to the real origin.

08
Phishing-resistant is the goal

FIDO2 / passkeys and security keys are origin-bound — they won't sign for a fake site:

key checks the URL→ wrong origin ⇒ no signature

Nothing to type, phish, or relay — CISA's recommended MFA.

09
The phishing-resistant factors
🔑
Security key (FIDO2) — a physical key you tap. Origin-bound, un-phishable.
📲
Passkey — a device-stored FIDO credential, unlocked by biometric or PIN.

Both bind the login to the real site — see the passkeys explainer for the full story.

11
Watch out
📶
SMS is the weakest factor — SIM-swap & interception. Use it only as a last resort.
🔢
Number matching beats blind push — context on the prompt stops fatigue-taps.
🍪
MFA isn't a cure-all — a stolen session token after login sails right past it.
🚪
Enrollment & recovery — reset flows & backup codes are prime attack paths.
12
The takeaway
MFA = two different factor types — and phishing-resistant factors (passkeys · security keys) beat codes and push.
Add a 2nd factor→ Make it a different type→ Go phishing-resistant→ Require it everywhere

Securing GenAI apps — the ten risks OWASP flags for anything that puts an LLM in the loop (prompts, RAG, and tool-calling agents)

01
Why LLM apps need their own list — the model reads attacker text as instructions

An LLM blends your instructions and any text it reads into one prompt — so a document, web page, or tool result can quietly steer it. That's why classic app-sec isn't enough.

One reference, ten IDs — the sections below spotlight the ones that bite hardest.

03
LLM01 · Prompt Injection — the #1 risk: untrusted input becomes instructions
💬
Direct — the user's own prompt jailbreaks the model.
🪤
Indirect — hidden text in a fetched page or file hijacks it.

Treat every retrieved token as untrusted — the model has no built-in “that's just data” instinct.

04
LLM06 · Excessive Agency

Injection is worst when the agent can act. Too much autonomy = too much blast radius:

🧰
Too many tools — it can delete, pay, or email when it only needed to read.
🔑
Too much permission — broad scopes, write access, prod credentials.
🤖
Too much autonomy — high-impact actions with no human check.

Least privilege for agents — narrow tools, tight scopes, human-in-the-loop.

05
New in 2025

LLM07 · System Prompt Leakage

Hidden instructions get coaxed out — never hide secrets there.

LLM08 · Vector & Embedding

RAG stores: poisoned docs, leaks & cross-tenant bleed.

LLM10 · Unbounded Consumption

Expanded from Model DoS — runaway cost & token drain.

06
LLM07 · System Prompt Leakage

Assume the system prompt will be extracted. The bug isn't the leak — it's putting secrets in it:

API keys· credentials· hidden rules

Enforce controls in code, not in the prompt.

07
LLM08 · Vector & Embedding

RAG adds a knowledge base — and a fresh attack surface:

☣️
Poisoned docs — malicious text ingested, then retrieved as “fact”.
🚪
Cross-tenant leak — weak filters return another user's chunks.

Partition, sign, and access-control the vector store.

08
LLM03 Supply Chain · LLM04 Poisoning
📦
Supply chain — untrusted models, datasets, adapters & plugins from the hub.
🧪
Data / model poisoning — tampered training or fine-tune data plants backdoors.

Vet provenance, pin versions, scan artifacts — an SBOM for models.

09
LLM05 Output · LLM09 Misinfo · LLM02 Disclosure
🧨
Improper output handling — piping raw output to a shell, SQL, or the DOM → RCE / XSS.
🌀
Misinformation — confident hallucinations; users overrely on them.
🔓
Sensitive disclosure — PII or secrets surface in a reply.

The model's output is untrusted input to everything downstream — validate it.

10
Defenses that cut across the list — a starter checklist
🚧
Least-privilege tools — narrow scopes, human approval for high-impact actions (LLM06).
🧱
Treat all input as untrusted — retrieved text is data, not orders (LLM01/08).
🧼
Validate output — encode / sandbox before shell, SQL, or HTML (LLM05).
🔐
Secrets out of prompts — enforce in code; redact PII (LLM07/02).
📋
Vet the supply chain — pin & scan models, data, plugins (LLM03/04).
⏱️
Rate & cost limits — quotas, timeouts, budget caps (LLM10).
11
The takeaway
GenAI apps inherit every classic risk plus one new one — the model treats any text it reads as instructions, so untrusted input plus real tools is where it breaks.
Untrusted input→ Least privilege→ Validate output→ Limit & monitor

ZTNA vs VPN

Zero Trust Network Access — grant a user one app, never the whole network (default-deny, and the app stays invisible to the internet)

01
The problem with the VPN — it drops you onto the whole network

A VPN grants network access — broad and flat. ZTNA grants application access — one app, default-deny.

02
App-level, not network-level
🌐
VPN — connects you to a subnet. You can reach anything routable on it.
🎯
ZTNA — a broker grants one specific app, and nothing else. Default-deny.

Access is to an application — the network is never handed over.

03
Verify every time

Not a one-time tunnel. Identity & device are re-checked per app, per session:

identity· device posture· context

Zero Trust applied to access — trust is earned per request, never assumed.

04
How it works — the broker verifies, then stitches an outbound-only tunnel

The app is never reached directly — the broker checks you, then relays the session to just that one app.

05
Apps go dark

Because the connector dials out, apps expose no public inbound ports:

🕳️
Nothing to scan — no listening port on the internet to find or attack.
🧵
Broker stitches it — it joins two outbound sessions; the app stays hidden.

A much smaller attack surface — you can't hit what you can't see.

08
Flavors — not all “ZTNA” is equal

Agent

Client app on the device. Fuller posture signals.managed endpoints

Agentless

Reach apps in a browser. No install — limited scope.web apps · BYOD

Connector

Service-initiated: it dials out. vs endpoint-initiated.app-side reverse-connect

Service-initiated keeps apps dark; agentless trades reach for coverage.

09
On the wire — a per-app, default-deny access policy
// access is granted to ONE app, gated on identity + device + context
policy: { app: "finance-portal", allow: "role:finance", require_mdm: true, min_os: "current" }
// broker decision — evaluated per session, never a standing tunnel
{ "user": "ada@corp", "device": "managed ✓ · disk-encrypted ✓", "decision": "ALLOW → finance-portal only" }
// everything else → DENY (no network route ever exists)
10
Watch out
💻
Unmanaged / BYOD — weak posture signals; scope these tighter or route via agentless.
📦
Over-broad app defs — a “/app/*” that maps a whole subnet quietly recreates the VPN.
🪪
Weak identity — ZTNA is only as strong as the IdP & device posture behind it.
🌐
Agentless limits — browser-only reach can't cover thick clients or deep posture.
11
The takeaway
Replace network access with per-app access — verify identity + device every time, and expose nothing to the internet.
Verify identity→ Check device→ Broker one app→ App stays dark

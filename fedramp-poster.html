01
What it is

FedRAMP — the Federal Risk and Authorization Management Program — standardizes how cloud services (CSPs) are security-assessed, authorized, and continuously monitored for use by US federal agencies.

"Do once, use many times" — one authorization can be reused by any agency, instead of every agency re-assessing the same cloud.

02
Why it exists
♻️
Kill duplicate work — without it, each agency separately vetted the same cloud. One assessment now serves many.
📏
One consistent bar — a shared baseline of controls, not a patchwork of per-agency rules.

Codified by the FedRAMP Authorization Act (2022) — no longer just policy.

03
The authorization path — build to a baseline, get assessed, get authorized, then never stop monitoring

A CSP builds to a baseline, a 3PAO independently tests it, an agency grants the ATO, it lands in the Marketplace for reuse — and then continuous monitoring never stops.

04
The impact baselines — from FIPS 199 impact on Confidentiality · Integrity · Availability

Higher impact = a bigger set of NIST 800-53 controls to implement and prove. Low · Moderate · High are the core tiers; Li-SaaS is a tailored path for lightweight low-impact SaaS.

05
Built on NIST

FedRAMP doesn't invent its own security — it applies the federal standard:

SP 800-53 controls+ RMF process

Same controls, same Risk Management Framework — FedRAMP is the cloud-specific way to apply them.

06
Who authorizes
🏛️
Agency ATO — a sponsoring agency reviews the package and grants the Authorization to Operate.
🔄
FedRAMP Board — replaced the old JAB provisional path; governs the program itself.

The old JAB (Joint Authorization Board) provisional route is retired.

07
Authorization isn't the finish line

An ATO must be maintained — continuous monitoring (ConMon) proves the cloud stays secure:

monthly ConMonvuln scansPOA&Ms for open itemsannual assessment

A POA&M — Plan of Action & Milestones — is the tracked, deadlined plan to close each open finding.

08
FedRAMP 20x — the modernization push · in progress, status as of mid-2026

20x trades static documentation for KSIs and automated OSCAL evidence, and moves toward certification classes (Class A pilot · B ≈ Low · C ≈ Moderate · D ≈ High, later). Mid-2026: the Phase 2 pilot completed (first pilot authorizations March 2026); Phase 3 wide adoption for Low/Moderate is rolling out through Q3–Q4 2026 — still stabilizing.

09
The key players
ActorRole

The 3PAO is independent — it tests, it doesn't authorize.

10
Watch out
🏋️
ConMon burden — monthly monitoring & POA&M upkeep is ongoing cost, not a one-time project.
🔗
Reuse ≠ auto-inheritance — an existing ATO speeds adoption, but the new agency still reviews & accepts the risk.
🚧
20x is still moving — treat mid-2026 KSI/class details as evolving; verify against current FedRAMP guidance.
11
The takeaway
FedRAMP is assess once, reuse everywhere — NIST controls, an independent 3PAO test, an agency ATO, and continuous monitoring that never stops — now modernizing toward automated evidence.
Baseline→ 3PAO→ ATO→ Marketplace→ ConMon↺

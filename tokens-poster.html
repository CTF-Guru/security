THREE TOKENS

ID · Access · Refresh — three tokens, three jobs, three lanes (mix them up and you build a security hole)
CTF Farooq Ali
· security explainer series
v2026.06
01
Three tokens, three jobs — who you are · what you can do · keep going

One sign-in can hand back all three — each proves a different thing and travels to a different place.

02
ID token — proves identity

A signed JWT that answers “who signed in?” — an OIDC thing, minted for your app.

aud = the client app· claims: sub · email

NEVER used to call an API — it's proof of identity, not a key to a resource.

03
Access token — grants API access

Sent to the API as Authorization: Bearer … — short-lived (minutes).

opaque OR a JWT· aud = the API

Bearer = whoever holds it, uses it. Guard it; keep the lifetime tight.

04
Refresh token — keeps the session alive

Long-lived. Presented at the /token endpoint to silently mint a new access token — no re-login.

🔒
Store securely — it's the master key to fresh access.
🔁
Rotate — issue a new one each use; it can be revoked.

06
The token lifecycle — sign in once, refresh silently, revoke to end it

The refresh token is the loop — steps 3→5 repeat until it's revoked or the user logs out.

07

Where each one goes
09
In one sentence each

🪪 ID

Read it to know who logged in.aud = your app

🎟️ Access

Attach it to call the API.Authorization: Bearer

🔄 Refresh

Trade it for a fresh access token.POST /token

Keep each token in its own lane and the model stays simple.

10
Watch out — the classic token mistakes
🚫
ID token to an API — its aud is your app, not the API; the API will (rightly) reject it.
🎭
Access token as identity — it proves access, not who the user is. Use the ID token to log in.
🥷
Refresh-token theft — rotate on every use and detect reuse: a replayed old token means it was stolen.
⏳
Access tokens living too long — a leaked one is valid until it expires. Keep it minutes, not days.
11
The takeaway
Three tokens, three jobs — identity (ID) · access (Access) · continuity (Refresh). Keep them in their lanes.
ID → the app· Access → the API· Refresh → /token only

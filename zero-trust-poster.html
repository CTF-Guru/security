ZERO TRUST

Never trust, always verify — the network perimeter is gone, so location grants no trust (NIST SP 800-207 · CISA Zero Trust Maturity Model)

01
The perimeter is gone — old castle-and-moat vs zero trust

Cloud, remote work, and SaaS erased the network edge — being “on the LAN” or on the VPN grants no implicit trust.

02
Three core principles — NIST SP 800-207

🔎 Verify explicitly

Authenticate & authorize on every access using all signals — identity, device, context.

🎚️ Least-privilege access

Grant the minimum needed, just-in-time. No standing, broad access.

💥 Assume breach

Act as if attackers are already inside — segment, monitor, and limit blast radius.

Trust is never granted by network location — it is earned, per request, and re-checked.

03
Verify on every request

Not a one-time login — each access recomputes trust from live signals:

identity+ device posture+ context

Context = location · risk score · time · behaviour → a fresh policy decision each time.

05
The decision flow — PDP evaluates policy, PEP enforces it, per session

Continuous & per-session — the same request later can be denied as risk changes.

06
Microsegmentation

No flat network. Each resource sits in its own tiny segment with its own policy:

per-resource segments→ no lateral movement

A breach is boxed in — one compromised host can't roam. Blast radius shrinks.

07
Least privilege, continuously
🎚️
Minimal & per-session — just enough access, just-in-time. No standing admin rights.
🔁
Re-evaluated — access is revisited as risk, device health, or context shifts.

A session trusted a minute ago can be dropped the moment signals turn risky.

08
Castle-and-moat vs Zero Trust — what changes
QuestionCastle-and-moatZero Trust
Trust from network locationInside = trustedNone — verify anyway

Blast radius of a breachWhole networkOne segment

09
Watch out
🛒
Not a product — Zero Trust is an architecture and a journey, not a box you buy.
🪪
Identity is the foundation — get strong auth & MFA right first; everything builds on it.
💻
Don't skip device & data — posture and data-level controls matter as much as identity.
🐢
Big-bang rollouts fail — go incrementally, pillar by pillar, highest-risk first.
10
A policy, in the abstract
// PDP decision — every signal must pass, every time
ALLOW if identity = verified + MFA && device = managed + healthy
  && context.risk < threshold && access = least-privilege
// otherwise → DENY (re-evaluated continuously, per session)
11
The takeaway
Trust nothing by default — verify identity, device, and context on every request, and assume breach.
Never trust→ Verify explicitly→ Least privilege→ Assume breach
